RAMKLEENER — MASTER CONTEXT & META v2.0

A pip-installable Python CLI tool that scans Windows/Mac/Linux for memory-bloating
background processes and kills them safely using a tiered protection model.

ONE-LINER
"Scans system processes, filters using a safe tiered model, and terminates known
memory-heavy background tasks with user confirmation — all from the terminal."

SAFETY MODEL
NEVER_KILL_CORE     — locked forever, OS critical
NEVER_KILL_DEFAULT  — protected by default, user can override via config
SAFE_TO_KILL        — known bloat, user can extend via config
UNKNOWN             — always skipped (safe by default)

CONFIG — ~/.ramkleener/config.json
{
  "user_protected": [],
  "user_kill_list": [],
  "threshold_mb": 100
}

KNOWN ISSUES (fix in Phase 3)
- browsers in SAFE_TO_KILL need rethinking (foreground vs background)
- .exe suffix stripping added in scanner (Windows psutil returns name.exe)
- standby flush requires Admin
- lists need tuning per real machine data

KEY RULES
- never kill unknown processes
- scan → display → confirm → clean (always in this order)
- threshold filtering is mandatory
- errors must never crash the program
- cleaner executes, never decides

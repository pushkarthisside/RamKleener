Safety Model — Three Tiers
NEVER_KILL_CORE      — locked forever, no one can edit this
                       universal Windows/OS critical processes only

NEVER_KILL_DEFAULT   — protected by default, user can remove via config
                       game anti-cheats, launchers, security tools etc.

SAFE_TO_KILL         — known bloat, killed if found running
                       user can add their own entries via config
Kill logic:
for each running process:
    if name in NEVER_KILL_CORE    → skip (locked, always protected)
    elif name in NEVER_KILL_DEFAULT + user_protected → skip (default protected)
    elif name in SAFE_TO_KILL + user_kill_list → kill it
    else                          → skip (unknown = safe by default)
NEVER_KILL_CORE examples (locked):
lsass, svchost, csrss, wininit, winlogon, dwm, explorer, smss, services, System
NEVER_KILL_DEFAULT examples (protected by default, editable):
vgc, vgtray, RiotClientServices, RiotClientUx, MsMpEng, NisSrv, Steam, steamwebhelper, EpicGamesLauncher, Battle.net
SAFE_TO_KILL examples (killed by default, editable):
chrome, msedge, msedgewebview2, OneDrive, GoogleUpdate, AdobeIPCBroker, compattelrunner, MusNotification, SpotifyWebHelper, DiscordCrashService

Config File — ~/.ramcleaner/config.json
json{
  "user_protected": ["your_vpn.exe", "your_anticheat.exe"],
  "user_kill_list": ["Spotify.exe", "Teams.exe"],
  "threshold_mb": 100
}

user_protected — added to NEVER_KILL_DEFAULT at runtime
user_kill_list — added to SAFE_TO_KILL at runtime
threshold_mb — only kill processes using more than this much RAM


Known Risks to Handle in Code

Permission denied — check for admin at startup, fail gracefully with clear message
OS-specific breakage — wrap Windows-only code in platform.system() == "Windows" checks
Over-aggressive cleaning — threshold_mb in config, only kill above X MB
Trust risk — implement --dry-run flag, shows what would be killed without doing it

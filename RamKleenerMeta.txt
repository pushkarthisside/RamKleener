SAFETY MODEL

NEVER_KILL_CORE

* locked forever
* critical OS processes

NEVER_KILL_DEFAULT

* protected by default
* user can override via config

SAFE_TO_KILL

* known bloat
* user can extend

LOGIC:

for each process:
if in NEVER_KILL_CORE → skip
elif in NEVER_KILL_DEFAULT + user_protected → skip
elif in SAFE_TO_KILL + user_kill_list → kill
else → skip

UNKNOWN = SAFE

PHASE 1 LEARNINGS (IMPORTANT)

* confirmation before kill is required
* grouping processes improves clarity
* threshold prevents over-aggressive kills
* admin check improves UX

KEY RULES

* never kill unknown processes
* always show before acting
* scan → display → confirm → clean
* threshold filtering is mandatory
* errors should not crash program
* execution logic must stay separate from filtering logic
